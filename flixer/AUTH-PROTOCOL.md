# Authentication Protocol

## Overview

Every API request to hexa.su must be signed with HMAC-SHA256. The signing key is generated by the WASM module (`get_img_key()`), and the signature covers the key, timestamp, nonce, and API path.

## Step 1: Time Synchronization

Before making any API requests, sync with the server clock:

```
GET https://themoviedb.hexa.su/api/time?t=<Date.now()>
→ { "timestamp": 1740000000 }  // Unix seconds
```

Calculate offset:
```javascript
const rtt = localTimeAfter - localTimeBefore;
const serverTimeMs = response.timestamp * 1000;
serverTimeOffset = serverTimeMs + (rtt / 2) - localTimeAfter;
```

Get server-synced timestamp for requests:
```javascript
function getTimestamp() {
  return Math.floor((Date.now() + serverTimeOffset) / 1000);
}
```

## Step 2: WASM Key Generation

```javascript
const loader = new FlixerWasmLoader();
await loader.initialize(wasmBytes);
const apiKey = loader.getImgKey();
// Returns a 64-character string — length is validated client-side
// If key.length !== 64, the client throws an error
```

The key is derived from the mocked browser environment (canvas fingerprint, screen resolution, platform, language, session ID, timezone offset). Consistent mocks produce consistent keys.

## Step 3: Nonce Generation

```javascript
const nonceBytes = crypto.randomBytes(16);
const nonce = nonceBytes.toString('base64')
  .replace(/[/+=]/g, '')
  .substring(0, 22);
```

22-character alphanumeric string derived from 16 random bytes.

## Step 4: HMAC-SHA256 Signature

```javascript
const message = `${apiKey}:${timestamp}:${nonce}:${apiPath}`;
const signature = createHmac('sha256', apiKey)
  .update(message)
  .digest('base64');
```

The message format is: `key:timestamp:nonce:path`
- `key` = the WASM-generated API key
- `timestamp` = server-synced Unix seconds
- `nonce` = 22-char random string
- `path` = API path (e.g., `/api/tmdb/movie/550/images`)

## Step 5: Client Fingerprint

```javascript
function generateClientFingerprint() {
  const fpString = `2560x1440:24:${userAgent.substring(0, 50)}:Win32:en-US:${timezoneOffset}:iVBORw0KGgoAAAANSUhEUgAAASwA`;
  let hash = 0;
  for (let i = 0; i < fpString.length; i++) {
    hash = (hash << 5) - hash + fpString.charCodeAt(i);
    hash &= hash;  // Convert to 32-bit integer
  }
  return Math.abs(hash).toString(36);
}
```

The fingerprint is a base-36 hash of a string combining:
- Screen resolution (`2560x1440`)
- Color depth (`24`)
- First 50 chars of User-Agent
- Platform (`Win32`)
- Language (`en-US`)
- Timezone offset
- A static PNG header fragment (`iVBORw0KGgoAAAANSUhEUgAAASwA`)

## Complete Request Headers

```http
X-Api-Key: <64_char_wasm_key>
X-Request-Timestamp: <server_synced_unix_seconds>
X-Request-Nonce: <22_char_nonce>
X-Request-Signature: <hmac_sha256_base64>
X-Client-Fingerprint: <base36_hash>
Accept: text/plain
Accept-Language: en-US,en;q=0.9
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ...
```

Note: The live client code does NOT send `Referer`, `sec-ch-ua`, or `sec-ch-ua-platform` headers. Our codebase sends these as extras — they appear to be tolerated but not required.

## Key Rotation

- The WASM-generated key appears stable for a given set of fingerprint inputs.
- The WASM loader should be re-initialized every ~30 minutes to avoid stale state.
- After 5 consecutive failures, the WASM state is force-reset.
- Time sync should be refreshed on each WASM re-initialization.
